01 자료구조 — 실제로 메모리에 뭐가 있나?

LLM 파일(예: llama-7b.bin)을 열면 거대한 float32 배열들입니다. 이것이 "파라미터"입니다.

C로 표현하면:

/* 하나의 Transformer 레이어 가중치 구조체 */
typedef struct {
    /* 어텐션 (Q, K, V, Output 행렬) */
    float *wq;   /* [d_model × d_model] */
    float *wk;   /* [d_model × d_model] */
    float *wv;   /* [d_model × d_model] */
    float *wo;   /* [d_model × d_model] */

    /* 피드포워드 (FFN) */
    float *w1;   /* [d_model × d_ff]   */
    float *w2;   /* [d_ff   × d_model] */

    /* Layer Normalization */
    float *gamma; /* [d_model] */
    float *beta;  /* [d_model] */
} TransformerLayer;

typedef struct {
    float *token_embedding; /* [vocab_size × d_model] */
    TransformerLayer layers[32];  /* N개 레이어 */
    float *lm_head;          /* [d_model × vocab_size] */
} LLM;

실제 크기 계산 (LLaMA-7B 기준):

// d_model = 4096, d_ff = 11008, layers = 32
vocab_size = 32,000

포인트: 7B 모델 파일 크기가 28GB인 이유. float32 = 4바이트. 70B 모델 = ~280GB. 그래서 GPU VRAM이 중요합니다.

핵심 연산: 행렬-벡터 곱 (Linear Layer)

/*
 * y = W * x + b  (선형 변환)
 * x: 입력 벡터   [d_in]
 * W: 가중치 행렬 [d_out × d_in]
 * y: 출력 벡터   [d_out]
 */
void linear(float *y, float *W, float *x,
              float *b, int d_out, int d_in) {
    for (int i = 0; i < d_out; i++) {
        y[i] = b ? b[i] : 0.0f;
        for (int j = 0; j < d_in; j++) {
            y[i] += W[i * d_in + j] * x[j];
        }
    }
}
/* 이 O(n²) 루프가 LLM의 90% 이상 연산
 * GPU는 이것을 수천 코어로 병렬화 */

활성화 함수: GELU (비선형성 추가)

/* GELU: 현대 LLM의 기본 활성화 함수
 * ReLU(x) = max(0,x) 의 부드러운 버전 */
float gelu(float x) {
    /* 근사 공식 */
    float c = 0.044715f;
    float inner = sqrtf(2.0f / M_PI)
                  * (x + c * x*x*x);
    return 0.5f * x * (1.0f + tanhf(inner));
}
/* 왜 필요? 비선형이 없으면
 * W3*(W2*(W1*x)) = (W3*W2*W1)*x
 * → 그냥 하나의 행렬로 축약 가능
 * → 레이어를 여러 개 쌓는 의미가 없음! */
핵심: 비선형 활성화 함수 없이는 아무리 레이어를 쌓아도 선형 변환 하나와 같습니다. GELU가 "표현력"을 만듭니다.

03 역전파 — 어떻게 학습하는가?

Chain Rule(연쇄 법칙)을 이용해 오류를 역방향으로 전파, 각 가중치가 오류에 얼마나 기여했는지 계산합니다.

CS 관점: Forward Pass = 함수 합성 f₃(f₂(f₁(x))). Backpropagation = 그 합성 함수의 편미분을 Chain Rule로 계산. 자동 미분(Autograd) 시스템이 이를 자동화합니다.

단순한 예: y = W·x, Loss = (y - target)²

dL/dW = dL/dy · dy/dW = 2(y - target) · xT 
/* Forward */
float y = dot(W, x);           // y = Wx
float loss = (y - target) * (y - target); // MSE

/* Backward (역전파) */
float dL_dy = 2.0f * (y - target); // loss의 y에 대한 미분
float dL_dW = dL_dy * x;           // y의 W에 대한 미분 = x

/* 가중치 업데이트 (Gradient Descent) */
float lr = 1e-4f;       // learning rate
W -= lr * dL_dW;         // 오류 방향 반대로 이동

다층 레이어의 Chain Rule:

/*
 * 레이어 3개: y = f3(f2(f1(x)))
 *
 * dL/dW1 = dL/dy · dy/df2 · df2/df1 · df1/dW1
 *          \_____Chain Rule 연쇄_____/
 *
 * 각 레이어는 forward 때 입력(x)을 저장해두고
 * backward 때 gradient 계산에 사용 */

/* 실제로는 PyTorch의 Autograd가 자동으로 처리
 * 개발자는 forward만 정의하면 됨 */

/* Cross-Entropy Loss (LLM의 실제 loss 함수) */
float cross_entropy(float *probs, int target) {
    /* 정답 토큰의 확률 로그값 (최대화 목표) */
    return -logf(probs[target] + 1e-8f);
}
/* "다음 토큰이 'X'일 때 모델이 X를 얼마나
 * 높게 예측했는가"를 손실로 측정 */
수치 안정성 이슈: 레이어가 깊어질수록 gradient가 0에 수렴(소실) 또는 폭발합니다. LayerNorm, Residual Connection, 주의 깊은 초기화로 해결합니다.

04 Transformer 전체 구조

2017년 "Attention is All You Need" 논문의 핵심. RNN/LSTM을 버리고 Attention만으로 구현.

왜 RNN을 버렸나?

RNN 문제점
  • 순차 처리 → GPU 병렬화 불가
  • Gradient 소실/폭발 (장거리 의존성 취약)
  • "나는 어제 서울에서... 먹었다" — 먼 앞 단어 기억 어려움
  • 시퀀스 길이 O(n) 시간
Transformer 해결
  • 모든 토큰 동시 처리 → 완전한 병렬화
  • 모든 토큰 쌍 직접 연결 (O(1) 거리)
  • Residual로 Gradient 안정
  • 다만 메모리 O(n²) — 컨텍스트 길이 제한 이유

GPT 스타일 Transformer (Decoder-only) 전체 흐름:

int tokens[] = {592, 214, 1034}; // 입력 3개 토큰
int T = 3, d = 4096;

/* Step 1: Token Embedding + Position Encoding */
float x[T][d];
for(int t=0; tadd(x[t], token_embed[tokens[t]],
            pos_embed[t], d);  // 위치 정보 합산
}

/* Step 2: N개 Transformer 블록 통과 */
for(int l=0; ltransformer_block(x, &layers[l], T, d);
}

/* Step 3: 마지막 토큰 위치의 벡터만 사용 */
float logits[vocab_size];
linear(logits, lm_head, x[T-1], NULL, vocab_size, d);

/* Step 4: Softmax → 다음 토큰 확률 */
float probs[vocab_size];
softmax(probs, logits, vocab_size);
Positional Encoding이 필요한 이유: Attention은 순서를 모릅니다. "고양이가 쥐를 잡았다"와 "쥐가 고양이를 잡았다"를 구분하려면 위치 정보를 명시적으로 주입해야 합니다. GPT는 학습 가능한 위치 임베딩, LLaMA는 RoPE(회전 위치 임베딩)를 사용합니다.
05 Self-Attention — 가장 중요한 연산

모든 토큰이 다른 모든 토큰을 "쿼리"하며 관계를 계산합니다. 데이터베이스의 퍼지 쿼리와 유사합니다.

DB 비유: SQL의 SELECT value FROM table WHERE key SIMILAR TO query와 같습니다. Query(질문), Key(인덱스), Value(실제 값)으로 소프트 검색(확률적 룩업)을 수행합니다.

Scaled Dot-Product Attention 수식과 구현:

 * K = X · Wk  : "내가 무엇인가?" (Key)

Multi-Head Attention — 여러 "관점"을 병렬로:

/* 헤드 수 h=8이면 d_k = d/h = 512/8 = 64
 * 각 헤드가 독립적으로 다른 관계 패턴 학습
 * 헤드1: 문법적 관계 / 헤드2: 의미적 관계 / 헤드3: 지시어 관계 ... */

void multi_head_attention(float *out, float *x,
                           TransformerLayer *L, int T, int d) {
    float head_outputs[N_HEADS][T][d/N_HEADS];

    /* 각 헤드 병렬 계산 (실제로 GPU에서 배치 처리) */
    for(int h=0; hfloat *Wq_h = L->wq + h*(d/N_HEADS)*d; // 헤드별 가중치 슬라이스
        single_head_attn(head_outputs[h], x, Wq_h, ...);
    }

    /* 헤드 결합: concat → 선형 변환 */
    float concat[T][d];
    concatenate(concat, head_outputs, N_HEADS);  // [T × d] 복원
    matmul(out, concat, L->wo, T, d, d);          // 최종 출력 변환
}

for step in range(max_steps):
    # 1. 데이터 배치 로드
    tokens = next(data_loader)  # [B, T]
    inputs  = tokens[:, :-1]   # [B, T-1]
    targets = tokens[:, 1:]    # [B, T-1] 다음 토큰

    # 2. Forward Pass
    logits = model(inputs)      # [B, T-1, vocab]
    loss = nn.cross_entropy(
        logits.view(-1, vocab_size),
        targets.view(-1))

    # 3. Backward Pass (autograd가 자동 계산)
    optimizer.zero_grad()
    loss.backward()           # 역전파

    # 4. Gradient Clipping (폭발 방지)
    nn.utils.clip_grad_norm_(
        model.parameters(), max_norm=1.0)

    # 5. 파라미터 업데이트
    optimizer.step()

Adam Optimizer — SGD보다 왜 잘 될까?

"""
SGD:  W = W - lr * gradient
Adam: gradient를 적응적으로 조정

Adam은 각 파라미터마다
- m: gradient의 지수이동평균 (모멘텀)
- v: gradient² 의 이동평균 (분산)
를 별도 추적

업데이트: W -= lr * m / (√v + ε)
→ 많이 변한 파라미터는 lr 감소
→ 거의 안 변한 파라미터는 lr 증가
→ 학습이 훨씬 안정적
"""

# Learning Rate Schedule (Cosine Decay)
def get_lr(step):
    # Warmup: 처음엔 lr 천천히 증가
    if step < warmup_steps:
        return max_lr * step / warmup_steps
    # Cosine decay: 이후 서서히 감소
    progress = (step-warmup_steps)/(max_steps-warmup_steps)
    return min_lr + 0.5*(max_lr-min_lr)*(
        1 + math.cos(math.pi * progress))
Autograd 마법: PyTorch는 forward pass를 기록하며 계산 그래프(DAG)를 구성합니다. loss.backward() 한 줄로 수백 개 레이어의 gradient를 자동 계산합니다. C로 직접 짜면 수만 줄이 필요합니다.

07 추론(Inference) — 배포 환경에서의 생성

학습이 끝난 모델을 실제 서비스할 때. KV Cache, 양자화, 배치 처리가 핵심입니다.

Autoregressive 생성 루프:

/* 토큰을 하나씩 생성하는 루프 */
int tokens[MAX_LEN] = {592}; // 초기 프롬프트
int n_tokens = 1;

while(n_tokens < MAX_LEN) {
    /* Forward Pass: 현재까지의 전체 시퀀스 */
    float probs[vocab_size];
    forward(probs, tokens, n_tokens, model);

    /* 다음 토큰 샘플링 */
    int next = sample(probs, temperature);

    /* EOS(문장 끝) 토큰이면 종료 */
    if (next == EOS_TOKEN) break;

    tokens[n_tokens++] = next;
}
/* 매 토큰마다 전체 시퀀스 재계산 → O(T²) 문제
 * → KV Cache로 해결! */

KV Cache — 재계산 방지 최적화:

/* 문제: 토큰 추가할 때마다
 * K, V를 전체 재계산하는 낭비
 *
 * 해결: 이전 K, V를 캐시에 저장
 * 새 토큰의 K, V만 계산해서 append */

float K_cache[MAX_LEN][d];  // 누적 저장
float V_cache[MAX_LEN][d];

for(int t=0; t/* 새 토큰의 K, V만 계산 */
    K_cache[t] = compute_K(x[t], Wk);
    V_cache[t] = compute_V(x[t], Wv);

    /* Attention은 캐시 전체와 계산 */
    scores = Q[t] · K_cache[0..t]ᵀ / √d_k;
    out[t] = softmax(scores) · V_cache[0..t];
}
/* O(T²) → O(T): 매우 큰 속도 향상
 * 대신 메모리 T × d × 2 × N_LAYERS 필요 */

양자화(Quantization) — 메모리 절약:

/* float32 → int8/int4로 압축
 * 정밀도 소폭 희생, 메모리·속도 대폭 개선 */

/* float32: 4 bytes per weight */
float32  W_fp32 = 0.3847f;  // 0011 1110 1100 0101 ... (32bit)

/* int8 양자화: 8 bytes → 75% 메모리 절약 */
float   scale  = max(abs(W_fp32)) / 127.0f;
int8_t  W_int8 = (int8_t)(W_fp32 / scale); // 49
float   W_deq  = W_int8 * scale;            // ≈ 0.3843 (근사값)

/*
 * 실용 수치:
 * 7B 모델 fp32:  28GB  → A100 필요
 * 7B 모델 int8:  7GB   → RTX 3080 가능
 * 7B 모델 int4:  3.5GB → RTX 3060 가능
 * → 로컬 LLM 실행의 핵심 기술 (llama.cpp)
 */

Flash Attention — 메모리 병목 해결:

/*
 * 문제: Attention Score 행렬 [T × T]을
 * GPU HBM(고대역폭 메모리)에 쓰고 읽는 게 병목
 * T=4096이면 4096² × 4byte = 67MB!
 *
 * Flash Attention (2022):
 * - SRAM(on-chip, 빠름)에 타일 단위로 처리
 * - T×T 행렬을 HBM에 저장하지 않음
 * - 메모리 O(T²) → O(T) 로 감소
 * - 속도 2~4배 향상
 *
 * 핵심 아이디어: 수치적으로 동일하지만
 * 메모리 접근 패턴을 타일화
 */

// PyTorch에서는 자동 적용:
with torch.backends.cuda.sdp_kernel(
        enable_flash=True):
    attn_out = F.scaled_dot_product_attention(Q, K, V)

09 현대 LLM의 개선 기술

기본 Transformer에서 현대 LLM(LLaMA, GPT-4)으로 발전하며 추가된 주요 기술들.

RoPE (Rotary Position Embedding)

절대 위치 대신 상대 위치를 Q·K 계산에 회전 행렬로 직접 인코딩.

/* 위치 t의 벡터를 회전 행렬로 변환
 * cos(t·θ) -sin(t·θ)   [q₂ₖ  ]
 * sin(t·θ)  cos(t·θ) × [q₂ₖ₊₁]
 * → Q·Kᵀ 내적이 상대 위치만 반영
 * → 긴 컨텍스트 외삽 가능 */

GQA (Grouped Query Attention)

Multi-Head의 KV 헤드 수를 줄여 KV Cache 메모리 절약.

/* MHA:  Q헤드=32, K헤드=32, V헤드=32
 * MQA:  Q헤드=32, K헤드=1,  V헤드=1
 * GQA:  Q헤드=32, K헤드=8,  V헤드=8  ← LLaMA3
 *
 * KV Cache 감소: 32배 → 4배 절약
 * 품질 손실 최소화 */

SwiGLU (FFN 활성화)

FFN에서 GELU 대신 게이팅 메커니즘 추가. LLaMA에 사용.

/* 일반 FFN: y = GELU(xW₁)W₂
 * SwiGLU:   y = (xW₁ ⊗ SiLU(xW₃))W₂
 *               ↑ 값     ↑ 게이트
 * 게이트가 정보 흐름 제어
 * 동일 파라미터 수 대비 성능 향상 */

MoE (Mixture of Experts)

FFN을 N개 전문가로 분리, 입력마다 Top-K 전문가만 활성화.

/* DeepSeek-V3: 전문가 256개, Top-8 선택
 * 활성 파라미터: 21B (총 671B 중)
 * → 연산량 = 21B 모델, 표현력 = 671B 모델
 *
 * Router: 어떤 전문가 선택할지 결정
 * 가중치 = softmax(x · W_router)[top-k] */
