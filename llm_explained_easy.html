🔢 AI의 세계엔 숫자만 있다

AI는 글자도, 의미도 모릅니다. 오직 숫자만 봅니다. 그럼 어떻게 언어를 이해할까요?

단어 → 숫자 벡터 변환

"고양이"라는 단어는 AI 눈에 이렇게 보입니다:

"고양이"
→
ID: 4521
→
[0.23, -1.4, 0.87, 0.12,
-0.55, 1.3, ... (4096개)]

단어 하나가 4096개의 숫자 리스트로 표현됩니다.
이 숫자들이 그 단어의 "의미"를 담고 있습니다.

의미가 숫자에 담기는 방법

✏️ 핵심 비유
"왕"과 "여왕"은 숫자 공간에서 가까이 위치합니다.
"왕 - 남자 + 여자 = 여왕"이 수학적으로 성립합니다.
의미가 비슷한 단어 → 비슷한 숫자 벡터
왕 ↔ 여왕
매우 가까움
고양이 ↔ 강아지
가까움
고양이 ↔ 자동차
멀음
핵심 포인트: AI는 "고양이"라는 글자를 이해하는 게 아닙니다. 4096개의 숫자 패턴이 다른 단어들의 숫자 패턴과 어떻게 관계되는지를 학습합니다. 이것이 "의미를 이해한다"는 것의 실체입니다.
⚡ 뉴런 — "중요도를 곱해서 더하기"

신경망의 핵심 연산은 딱 하나입니다. 입력에 중요도(가중치)를 곱해서 전부 더하는 것.

뉴런이 하는 일 — 예시

"이 동네 치킨집 갈까?" 결정을 내리는 뇌세포를 상상해보세요.

🍗 맛있나
× 0.9
💰 가격
× 0.6
📍 거리
× 0.3
⏰ 웨이팅
× -0.5
합계 = (맛×0.9) + (가격×0.6) + (거리×0.3) + (웨이팅×-0.5)
합계 > 임계값 → "가자!" 신호 발생

인공 신경망도 똑같습니다

입력 숫자들에 각각 "중요도(가중치)"를 곱하고 전부 더합니다.
이 "가중치"가 바로 AI가 학습하는 것의 정체입니다.

GPT-4의 파라미터 수조 개 = 이 가중치 숫자들의 개수
학습 = 이 가중치들을 조금씩 조정하는 과정

📊 엑셀 비유
신경망 연산은 사실 엑셀 표의 SUMPRODUCT 함수입니다.
=SUMPRODUCT(입력범위, 가중치범위)
이걸 수십억 번 반복하면 AI입니다.
왜 여러 층(레이어)을 쌓나?
1층: 선, 점 같은 단순 패턴 인식
2층: 눈, 코 같은 복합 패턴
3층: 얼굴 전체
층이 깊어질수록 더 추상적인 개념 학습 가능

신경망 구조 — 층을 쌓는 이유

🔤 토큰 — AI가 언어를 자르는 방식

AI는 글자 단위도, 단어 단위도 아닌 "토큰" 단위로 언어를 처리합니다.

같은 문장을 3가지 방식으로 나누면

❌ 글자 단위 (비효율)
인공지능은 미래다
9개 → 너무 세밀, 학습 느림
❌ 단어 단위 (신조어 문제)
인공지능은미래다
2개 → "챗GPT" 같은 신조어 처리 불가
✅ 토큰 단위 (최적화)
인공지능은 미래다
5개 → 의미 단위로 최적 분할

🏋️ 학습 — 틀리면 수정, 반복

AI 학습은 시험을 무한 반복하며 오답을 고치는 과정입니다.

🎯 수능 공부 비유
학생(AI)이 모의고사(다음 단어 예측)를 보고 채점지(정답)와 비교합니다.
"파이썬의 장점은 ___" → AI 예측: "빠른" → 정답: "가독성"
→ 틀렸으니 "가독성"의 가중치를 높이는 방향으로 수정
→ 이걸 수천억 번 반복 → 지능이 생김
1

예측 (Forward)

입력 → 신경망 통과 → 다음 단어 확률 출력.
"오늘 날씨가" → ["맑다" 40%, "춥다" 35%, ...]

2

채점 (Loss 계산)

정답과 비교해 "얼마나 틀렸나" 점수 계산.
정답이 "맑다"인데 "춥다"를 높게 예측했다면 높은 오류

3

책임 추적 (Backward)

오류가 어느 가중치 때문인지 역방향으로 추적.
"춥다를 높인 건 3번 레이어 542번 가중치 때문이야"

4

수정 (Gradient Descent)

책임 있는 가중치를 아주 조금 조정.
조정량은 0.0001처럼 미세하게. 이걸 수조 번 반복.

학습 단계별 소요 비용

역전파(Backpropagation)란?
오류의 "책임"을 뒤로 전파하는 알고리즘. 마치 팀 프로젝트에서 실패 원인을 각 팀원에게 배분하는 것처럼, 각 가중치가 오류에 얼마나 기여했는지 계산합니다. 수학적으로는 Chain Rule(연쇄 법칙)을 적용합니다.
🔦 어텐션 — 형광펜으로 중요한 곳에 집중

2017년 구글이 발명한 어텐션 메커니즘. 현대 AI의 핵심입니다.

🖊️ 형광펜 비유
"그 고양이가 생선을 먹었다. 그것은 배가 불렀다."

"그것"이 무엇인지 파악하려면 앞 문장을 전부 다시 읽으며 "고양이"에 형광펜을 칩니다.
어텐션 메커니즘이 하는 일이 바로 이겁니다 — 어떤 단어에 집중할지 확률로 계산.

→ "그것" = "고양이" (72% 집중) 로 올바르게 파악

어텐션이 필요한 이유

❌ 어텐션 없이

"나는 어제 서울에서 맛있는 음식을 먹었다"
→ "먹었다"와 "나는"의 연결이 끊어짐 (문장이 길수록 심각)

✅ 어텐션 있으면

모든 단어가 다른 모든 단어를 직접 참조 가능.
아무리 긴 문장도 관계 파악 가능.

🗃️ 데이터베이스 비유
어텐션 = 퍼지 검색(Fuzzy Search)
Query: "그것이 가리키는 것은?"
Key: 각 단어의 "나는 무엇인가"
Value: 각 단어의 실제 내용
→ 가장 관련 높은 Value를 가중 합산
멀티헤드 어텐션이란? 형광펜을 여러 색으로 동시에 칩니다. 파란 형광펜은 문법 관계, 빨간 형광펜은 의미 관계, 초록 형광펜은 지시어 관계를 각각 담당합니다. GPT-4는 이런 형광펜을 동시에 128개 사용합니다.
