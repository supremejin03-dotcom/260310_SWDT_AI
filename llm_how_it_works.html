📜 LLM의 탄생 — 60년 역사

오늘날의 ChatGPT, Claude는 하루아침에 나온 게 아닙니다. 60년간 수많은 연구자들의 실패와 도전이 쌓인 결과입니다.

1943
인공 뉴런의 탄생 — McCulloch & Pitts
수학자와 신경과학자가 뇌 뉴런을 수학으로 표현. "0 또는 1을 출력하는 논리 게이트"로 뉴런을 모델링. AI의 씨앗이 뿌려짐.
1958
퍼셉트론 — Frank Rosenblatt
최초의 "학습하는" 기계. 입력에 가중치를 곱해 합산, 임계값 초과 시 1 출력. 이미지 분류 시연으로 전 세계가 열광했지만 XOR 문제로 한계 드러남.
1986
역전파(Backpropagation) — Rumelhart, Hinton
신경망이 "틀렸을 때 어떻게 수정할지" 알게 됨. 오류를 뒤로 전파하며 가중치 조정. 다층 신경망 학습이 가능해짐. AI 부활의 시작.
1997
LSTM — Hochreiter & Schmidhuber
순서가 있는 데이터(문장, 음성)를 기억하는 신경망. "어제 먹은 것"을 기억하는 단기 기억 회로. 언어 모델의 첫 번째 도약.
2012
딥러닝 혁명 — AlexNet, GPU 활용
Geoffrey Hinton의 팀이 이미지 인식 대회를 압도적 차이로 우승. GPU로 병렬 계산, 대규모 데이터 학습 가능해짐. 딥러닝 시대 개막.
2017
"Attention is All You Need" — Google Brain
트랜스포머 아키텍처 논문 발표. 8명의 구글 연구자가 기존 방식을 완전히 뒤엎음. "어텐션 메커니즘"으로 장거리 의존성 문제 해결. 현대 LLM의 원형.
2018–2019
BERT, GPT-1, GPT-2 — Google & OpenAI
대규모 텍스트로 사전학습(Pre-training) 개념 정착. BERT는 문맥 이해, GPT는 텍스트 생성에 특화. GPT-2는 "너무 위험하다"며 공개 보류 논란.
2020
GPT-3 — 1750억 파라미터
코딩, 번역, 요약을 별도 학습 없이 수행. "In-context Learning" 개념 등장. 세상이 AI를 다시 보기 시작함.
2022–현재
ChatGPT, Claude, Gemini — 대중화
RLHF(인간 피드백 강화학습)로 대화형 AI 완성. ChatGPT 출시 5일 만에 100만 사용자. Claude, Gemini, LLaMA 등 경쟁 가속화. AI 시대 본격 개막.
🧠 뇌를 본뜬 신경망

AI 신경망은 인간의 뇌에서 영감을 받았습니다. 하지만 어디까지 닮았고 어디서 다를까요?

🧬 생물학적 뇌
• 뉴런 약 860억 개
• 시냅스 연결 100조 개
• 전기화학 신호 전달
• 경험으로 연결 강도 변화
• 소비 전력 약 20W
• 평생 학습 가능
💻 인공 신경망 (LLM)
• 노드(뉴런) 수백만~수십억 개
• 파라미터 수십억~수조 개
• 숫자(부동소수점) 계산
• 역전파로 가중치 조정
• 소비 전력 수백~수천W
• 학습 후 고정 (추론 시)

🔬 뉴런 vs 인공 노드 비교

핵심 아이디어: 뉴런은 여러 입력 신호를 받아 합산하고, 임계값을 넘으면 다음 뉴런으로 신호를 보냅니다. 인공 노드는 이를 수학적으로 흉내냅니다. 입력값 × 가중치를 모두 더하고, 활성화 함수로 출력값을 결정합니다.
⚡

활성화 함수

뉴런이 "발화할지 말지" 결정하는 함수. ReLU, Sigmoid, GELU 등. 비선형성을 추가해 복잡한 패턴 학습 가능.

🔗

레이어 (층)

뉴런들을 층으로 쌓음. 입력층 → 은닉층(여러 개) → 출력층. GPT-4는 약 120개 레이어를 사용.

🎯

파라미터 (가중치)

학습을 통해 조정되는 숫자들. GPT-3는 1750억 개, GPT-4는 수조 개 추정. 더 많을수록 더 많은 지식 저장.

🔤 토큰 — AI가 언어를 보는 방식

AI는 글자가 아닌 "토큰" 단위로 언어를 처리합니다. 이 차이가 왜 중요한지 알아봅니다.

문장이 토큰으로 쪼개지는 과정

아래 문장은 AI에게 이렇게 보입니다:

"인공지능은 미래를 바꿀 것입니다"
인공 지능 은 미래 를 바꿀 것 입니다

→ 8개 토큰. 영어는 약 4글자 = 1토큰, 한국어는 1~2글자 = 1토큰

왜 토큰으로 나눌까?

  • 알파벳/글자 단위는 너무 세밀 → 학습 비효율
  • 단어 단위는 새 단어(신조어) 처리 불가
  • 토큰은 의미 있는 최소 단위로 최적화
  • 어휘 사전 크기 관리 가능 (보통 5~10만 개)

토큰 → 숫자 → 벡터

① 토큰 → 고유 번호 (예: "인공" = 15234)
② 번호 → 임베딩 벡터 (수백 개 숫자로 의미 표현)
③ 벡터들이 신경망을 통과하며 의미 파악
④ 최종 출력: 다음 토큰의 확률 분포
임베딩의 마법: "왕 - 남자 + 여자 = 여왕"이 수학적으로 성립합니다. AI는 단어를 수백 차원의 공간에 배치하고, 의미적으로 비슷한 단어는 가까운 위치에 놓습니다.
🏋️ 어떻게 학습하는가?

LLM 학습은 3단계로 이루어집니다. 방대한 텍스트를 읽고, 예측하고, 틀리면 수정하는 반복 과정입니다.

1

데이터 수집 (인터넷 텍스트)

웹 페이지, 책, 논문, 코드, 위키피디아 등 수십억 페이지의 텍스트를 수집. GPT-4 학습 데이터는 수조 토큰.

2

다음 단어 예측 (사전학습)

"오늘 날씨가 ___" → 모델이 확률을 예측. 정답과 비교해 오류를 계산.

3

역전파 (Backpropagation)

오류를 뒤로 전파하며 파라미터(가중치) 조금씩 수정. 이 과정을 수천억 번 반복.

4

RLHF — 인간 피드백 강화학습

사람이 AI 응답을 평가 → 좋은 응답은 강화, 나쁜 응답은 억제. ChatGPT, Claude가 대화형이 된 이유.

학습 데이터 구성 (추정)

핵심 원리 — 다음 토큰 예측: LLM은 본질적으로 "다음 단어가 뭘지 맞추는" 게임을 수천억 번 반복합니다. 그 과정에서 문법, 사실, 논리, 코딩까지 자연스럽게 학습합니다. 이것이 "언어 모델"의 본질입니다.

⚙️ 트랜스포머 — 현대 LLM의 심장

2017년 구글이 발표한 "Attention is All You Need" 논문이 AI를 바꿨습니다. 핵심은 "어텐션" 메커니즘입니다.

어텐션이란?
"은행에 가서 돈을 뽑았다" → "은행"의 의미를 파악할 때 "돈"과 "뽑았다"에 집중(어텐션). 문장 내 모든 단어의 관계를 계산.
멀티헤드란?
여러 개의 어텐션 헤드가 동시에 서로 다른 관계를 봄. 어떤 헤드는 문법, 어떤 헤드는 의미, 어떤 헤드는 지시어 관계를 담당.
위치 인코딩이란?
트랜스포머는 순서를 모름. 그래서 각 토큰에 위치 정보(1번째, 2번째...)를 수학적으로 더해줌. 사인파 함수 사용.
피드포워드 레이어란?
어텐션 이후 각 토큰을 독립적으로 처리. 학습된 지식의 대부분이 여기 저장됨. "AI의 장기 기억 창고."

어텐션 시각화 — "그 고양이가 자리를 잡았다. 그것은..."

"그것"이라는 단어를 처리할 때 모델이 어떤 단어에 집중하는지 (밝을수록 높은 어텐션)

💬 추론 — AI가 답변을 만드는 과정

학습이 끝난 모델이 실제로 답을 생성할 때 어떻게 동작할까요? 토큰을 하나씩 선택하는 과정입니다.

Temperature란?

🌡️ Temperature = 0
항상 최고 확률 토큰 선택. 결정론적, 예측 가능. 코딩에 적합.

🎲 Temperature = 1
확률 분포 그대로 샘플링. 창의적이지만 가끔 엉뚱함.

🔥 Temperature = 2
분포를 평평하게 만들어 더 랜덤. 창작, 브레인스토밍에 활용.
토큰 하나씩 생성: AI는 답변을 한 번에 만들지 않습니다. "파이썬의 장점은" → " 가독성" → "이" → " 매우" → " 높습니다" → "." 이렇게 토큰 하나씩 선택하며 쌓아갑니다. 그래서 스트리밍으로 글자가 하나씩 나오는 것처럼 보이는 거죠.

파라미터 수 역사적 증가 추이

🔮

MoE (전문가 혼합)

파라미터 전부를 쓰지 않음. 입력에 따라 적절한 "전문가" 모듈만 활성화. GPT-4, DeepSeek이 사용. 효율 대폭 향상.

📏

컨텍스트 윈도우

한 번에 처리할 수 있는 토큰 수. Claude는 200K 토큰(책 1~2권). Gemini는 최대 1M 토큰. 길수록 긴 문서 처리 가능.

🛠️

멀티모달

텍스트뿐 아니라 이미지, 음성, 영상까지 처리. GPT-4o는 실시간 음성 대화. Claude는 이미지 분석. AI의 감각이 확장 중.

⚠️ LLM의 한계와 미래

LLM이 강력하지만, 알아야 할 본질적 한계가 있습니다.

현재 한계

🎭 환각 (Hallucination)
없는 사실을 자신 있게 말함. "다음 토큰 예측" 기반이라 틀려도 그럴듯한 문장 생성. 항상 검증 필요.
📅 지식 컷오프
학습 데이터 이후 사건을 모름. RAG(검색 증강 생성)으로 보완 중. 실시간 정보는 별도 도구 필요.
🔢 수학·논리 약점
복잡한 계산 오류 발생. 언어 기반이라 정확한 산술 어려움. Chain-of-Thought으로 개선 중.
🔒 컨텍스트 제한
대화가 길어지면 앞 내용을 잊음. 컨텍스트 윈도우를 넘으면 이전 정보 손실.

미래 방향

🤖 AI 에이전트
LLM이 도구를 사용하고 계획을 세워 스스로 일 처리. Claude Code, Cursor 같은 AI 코딩 에이전트.
💭 추론 모델
답하기 전에 "생각하는 시간"을 가짐. OpenAI o1, Claude 3.7 Sonnet (Extended Thinking). 복잡한 문제 해결력 대폭 향상.
🌐 멀티에이전트
여러 AI가 서로 소통하며 협력. 기획 AI, 코딩 AI, 검증 AI가 팀을 이룸. 인간 팀처럼 복잡한 프로젝트 수행.
🧬 소형화 & 온디바이스
스마트폰, PC에서 직접 실행. Apple Intelligence, Gemini Nano. 클라우드 없이 개인 AI 가능.
📌 핵심 정리
🧠
뇌에서 영감
뉴런 → 인공 노드, 시냅스 → 가중치. 구조만 닮았고 동작은 다름.
🔤
토큰 단위 처리
글자 → 토큰 → 숫자(임베딩) → 벡터 공간에서 의미 계산.
🏋️
예측+수정 반복
다음 단어 예측 → 틀리면 역전파 → 수천억 번 반복 = 지능.
⚙️
어텐션이 핵심
단어 간 관계를 계산. 문맥을 이해하는 원동력.
🎲
토큰 하나씩 생성
답변을 한 번에 만들지 않음. 확률적으로 토큰 선택해 쌓음.
