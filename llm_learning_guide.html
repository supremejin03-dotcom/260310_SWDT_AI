📖 PART 1

LLM이 뭔가요?

ChatGPT, Claude, Gemini… 다 같은 계열입니다. 한번에 이해해봅시다.

🤖

LLM = Large Language Model

인터넷의 방대한 텍스트(책, 논문, 코드, 웹페이지 등)를 학습해서 사람처럼 글을 읽고 쓰고 대화할 수 있게 만든 AI 모델입니다.

쉽게 말하면 — 세상의 거의 모든 지식을 읽은 뒤 "이 질문엔 이런 답이 적절하다"를 확률적으로 추론하는 엔진입니다.

⏱️

즉각적인 피드백

답 기다릴 필요 없음. 코드를 작성하면 바로 리뷰, 오류가 나면 바로 해설.

🎯

맞춤형 설명

"중학생에게 설명하듯", "비유로", "코드 예시와 함께" — 내가 원하는 방식으로.

♾️

무한 반복

같은 질문을 100번 해도 됩니다. 눈치 없고 지치지 않는 과외 선생님.

🧩

연결 학습

"이 개념이 실제로 어디서 쓰이나요?" — 추상 지식을 실전과 바로 연결.

📝

문서·요약 생성

긴 문서, 강의 내용을 붙여 넣으면 핵심만 요약·정리해 줌.

🧠

소크라테스식 대화

"정답 말고 힌트만 줘" — 직접 생각하게 유도하는 대화형 학습 가능.

나의 질문

AI: 거울 두 개를 마주보게 놓으면 그 안에서 무한히 반복되는 이미지 보신 적 있죠? 재귀함수가 바로 그겁니다...

💡 "다시 설명해줘", "예시 하나 더 줘"를 두려워 말 것
📋 모드 2 · 커리큘럼 설계
나의 질문
나: 나는 파이썬을 전혀 모르는 비개발자야. 3개월 안에 데이터 분석을 할 수 있을 정도가 목표야. 주간 학습 계획 만들어줘.

AI: 1주차: 변수·자료형·조건문 / 2주차: 함수·반복문...

💡 목표, 현재 수준, 가능한 시간을 함께 알려줄수록 정확함
🧪 모드 3 · 퀴즈·자기테스트
나의 질문
나: 방금 CSS 플렉스박스 공부했어. 내가 제대로 이해했는지 확인할 수 있게 퀴즈 5개 내줘. 정답은 내가 먼저 답하고 나면 알려줘.

AI: Q1. `justify-content: center`는 어느 방향으로 정렬하나요?

💡 배운 직후 퀴즈 = 기억 정착률 2배 (간격 반복 학습 원리)

나의 질문

[코드 붙여넣기]

AI: 3가지 개선점이 있습니다. 첫째, 변수명이 너무 짧아서...

🌟

소크라테스 모드 — 가장 강력한 학습법

"정답을 바로 알려주지 말고, 내가 스스로 답에 도달할 수 있게 질문으로 유도해줘"라고 말하면 됩니다.
AI가 힌트를 주고, 반론을 던지고, 생각을 확장시켜 줍니다. 단순히 답을 받는 것보다 기억에 훨씬 오래 남습니다.

💬 PART 4

바로 쓸 수 있는 프롬프트 모음

복사해서 바로 ChatGPT / Claude에 붙여넣으세요.

상황프롬프트 예시효과
개념 이해 "[개념]을 처음 배우는 사람에게 현실 세계 비유로 설명해줘. 예시 코드도 포함해서." 직관적 이해
학습 계획 "나는 [현재 수준]이고 [목표]를 [기간] 안에 이루고 싶어. 주차별 커리큘럼 만들어줘." 구조화된 학습
퀴즈 "방금 [주제] 공부했어. 5지선다 퀴즈 5개 만들어줘. 내가 답하면 채점해줘." 기억 정착
코드 리뷰 "이 코드의 문제점과 개선 방향을 주니어 개발자 관점에서 설명해줘. [코드]" 실력 향상
오류 해결 "이 에러가 왜 났는지, 어떻게 고치는지 알려줘. [에러 메시지 붙여넣기]" 빠른 디버깅
심화 탐구 "[개념]을 완전히 이해했어. 이게 실무에서 어떻게 쓰이는지, 연관 개념은 뭔지 알려줘." 깊이 있는 이해
요약 정리 "[긴 글 / 강의 내용 붙여넣기]. 핵심 포인트 5개로 요약하고 내가 꼭 기억해야 할 것 강조해줘." 정보 소화
소크라테스 "[주제]를 가르쳐줄 건데, 정답은 알려주지 말고 질문으로 나를 유도해줘. 내가 스스로 발견하게." 사고력 향상
⚠️ PART 5

이것만은 주의하세요

LLM을 잘못 쓰면 오히려 독이 됩니다. 실사용자들이 많이 빠지는 함정들입니다.

❌ 함정 1 · 답만 받고 이해 안 하기 "코드 짜줘" → 받고 끝. 왜 그렇게 짰는지 모르면 다음에 또 물어봐야 합니다.
✅ 해결: "이 코드 각 줄이 왜 이렇게 작성됐는지 설명해줘"
❌ 함정 2 · AI 말을 무조건 믿기 LLM은 그럴 듯하게 틀립니다 (할루시네이션). 특히 최신 정보, 숫자, 법률은 반드시 검증하세요.
✅ 해결: "이 정보 신뢰할 수 있어? 출처가 있어?" 항상 의심
❌ 함정 3 · 너무 긴 대화 한 번에 처리 세션이 길어질수록 AI가 앞 내용을 "까먹고" 이상한 답을 냅니다.
✅ 해결: 주제별로 새 대화 시작. 핵심 내용은 직접 정리
❌ 함정 4 · 학습 없이 AI에 의존 AI가 다 만들어 준다고 이해 없이 쓰면 응용이 안 됩니다. 기초 개념은 내 것으로 만들어야 합니다.
✅ 해결: AI는 선생님, 나는 학생. 배우는 과정을 생략하지 말 것
❌ 함정 5 · 모호한 질문 "파이썬 알려줘" → 너무 광범위. AI도 어디서부터 말해야 할지 모릅니다.
✅ 해결: 현재 수준 + 목표 + 구체적 상황을 함께 제시
💡 핵심 원칙: AI는 도구, 두뇌는 내 것 LLM은 학습을 가속하는 도구입니다. 이해·판단·응용은 여전히 내가 해야 합니다. AI가 가르쳐준 내용을 직접 손으로 써보고, 설명해보고, 변형해보는 것이 진짜 학습입니다.

LLM 학습 로드맵

오늘부터 시작할 수 있는 단계별 가이드입니다.

1
🟢 오늘 당장 · 무료

AI와 첫 대화 시작

  • 🌐
    ChatGPT.com 또는 Claude.ai 접속
    계정 만들고 무료로 사용 시작. 지금 당장 궁금한 것 아무거나 질문해보기
  • 🎯
    "나를 소개"하고 맞춤 학습 요청하기
    현재 직업, 목표, 학습 가능 시간을 AI에게 알려주면 개인 맞춤 계획을 만들어 줌
  • ✍️
    배운 내용 직접 손으로 정리
    노션, 옵시디언, 종이 노트 상관없음. AI가 가르쳐준 내용을 내 말로 다시 써보기
2
🔵 1~2주 · 무료

나만의 학습 루틴 만들기

  • 📅
    매일 30분 AI와 공부 세션
    개념 1개 → 질문 → 이해 → 퀴즈 → 정리. 짧고 꾸준하게가 핵심
  • 🧪
    배운 것 바로 실습
    코딩이면 직접 타이핑, 글쓰기면 직접 작성. "이해"와 "할 수 있음"은 다름
  • 💬
    모르는 것이 생기면 즉시 질문
    "나중에 찾아볼게"가 아니라 그 자리에서 바로. 흐름이 끊기지 않게
3
🟣 2~4주

프롬프트 수준 올리기

  • 🎭
    역할 지정 프롬프트 활용
    "너는 10년 경력 파이썬 개발자야. 주니어 개발자인 나에게 코드 리뷰해줘"
  • 🔗
    System Prompt 직접 설정
    Claude Projects / ChatGPT Custom Instructions에 내 수준·목표·규칙을 미리 입력해두면 매번 설명 안 해도 됨
  • 📊
    내 학습 데이터 분석 요청
    "지난 2주 동안 내가 틀렸던 개념들을 보니 패턴이 뭔지 분석해줘"
4
🟡 1~2개월

실전 프로젝트로 검증

  • 🏗️
    배운 내용으로 실제 뭔가 만들기
    아주 작은 것도 OK. "동작하는 무언가"를 만들면 학습이 통합됨
  • 🤖
    AI를 코딩 파트너로 활용
    Cursor, Claude Code 등 에이전트 도구로 실제 프로젝트 진행해보기
  • 📝
    배운 것을 남에게 설명해보기
    블로그, 커뮤니티 글쓰기, 혹은 AI에게 "내가 이 개념 제대로 이해했는지 내 설명 평가해줘"
5
🌸 진행형

멀티 LLM 전략으로 확장

  • 🔀
    모델을 목적별로 분리해서 사용
    코딩·깊이 있는 분석 → Claude / 빠른 검색·최신 정보 → Perplexity / 이미지 → Gemini
  • 🧩
    NotebookLM 활용
    Google의 무료 도구. PDF·유튜브 영상을 올리면 AI가 읽고 질문에 답해주는 개인 지식베이스 구축
  • 🔄
    AI에게 배운 것을 AI로 자동화
    반복 업무를 스크립트로 만들어 자동화. 배운 코딩 지식이 실제 생산성으로 이어짐

LLM은 지식을 대신해주지 않습니다

LLM은 학습의 속도를 높여주는 엔진이지, 학습 자체를 대신해 주는 도구가 아닙니다.
이해하고, 손으로 써보고, 틀리고, 다시 묻고 — 그 과정을 AI와 함께 하면
혼자 공부할 때보다 3~5배 빠르게 성장할 수 있습니다.
